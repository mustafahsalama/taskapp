## Features

- **Add, edit, delete** tasks with title, description, and due date
- **Toggle completion** — tasks animate between Pending and Completed sections
- **Persistent storage** via [Hive](https://pub.dev/packages/hive) (survives app restarts)
- **Smart sorting** — pending tasks sorted by due date, completed tasks by completion time
- **Due date colour coding** — overdue (red), today (orange), tomorrow (amber), future (green)
- **Swipe to delete** with confirmation dialog
- **Animations** — entry slide/fade per tile, checkbox bounce, FAB elastic scale, collapse/expand completed section
- **Error handling** via [dartz](https://pub.dev/packages/dartz) `Either<Failure, T>` with optimistic UI updates and automatic rollback on failure

```
├── data/

├── bloc/

│       │                           TaskDeleted · TaskToggleCompleted

│
└── widgets/

```

### State flow

```
UI Event
  └─► TaskBloc.add(event)
        ├─ Optimistic emit (instant UI update)
        ├─ await TaskRepository (Hive I/O)
        │    └─ Either<Failure, Unit>
        ├─ Right(_) → keep optimistic state
        └─ Left(failure) → rollback + emit TaskFailureState
                               └─► BlocConsumer.listener → SnackBar
```

## Getting Started

### Prerequisites

- Flutter SDK `>=3.7.2`
- Dart SDK `>=3.7.2`

### Run

```bash
# 1. Install dependencies
flutter pub get

# 2. Generate Hive adapter (only needed after changing Task model)
dart run build_runner build --delete-conflicting-outputs

# 3. Run the app
flutter run
```

### Regenerate Hive adapter

Any time you add or change a `@HiveField` on `Task`, re-run:

```bash
dart run build_runner build --delete-conflicting-outputs
```

## Key Design Decisions

**Singleton repository**
`TaskRepository` uses a private constructor and a `static final instance` field. There is one Hive box, one data source — a singleton is the right fit. The BLoC constructor accepts an optional `repository` parameter for testing.

**dartz `Either` — hide `Task` name clash**
`dartz` exports its own `Task` type. The repository imports dartz with `hide Task` to avoid the ambiguity:
```dart
import 'package:dartz/dartz.dart' hide Task;
```

**Optimistic updates**
Every mutation emits the new state immediately before the async Hive write. If the write fails, the previous state is re-emitted (rollback) and a `TaskFailureState` is emitted to trigger a SnackBar.

**Task extends Equatable, not HiveObject**
`HiveObject` is only needed for calling `task.save()` / `task.delete()` directly on a model. Since all Hive access goes through `TaskRepository`, `Task` can extend `Equatable` instead, enabling value-based equality in BLoC states and list comparisons.

**Form screen stays BLoC-free**
`TaskFormScreen` returns a `Task` via `Navigator.pop`. `HomeScreen` dispatches the appropriate event (`TaskAdded` / `TaskUpdated`). This keeps the form a pure data-entry component, independent of state management.
